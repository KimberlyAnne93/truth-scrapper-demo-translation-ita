For point and click skipping:

I suggest you include some basic code to hide the skipping arrow/prevent starting to skip while on the point and click screens. You can do this via adding the line `use prevent_skipping()` e.g.

screen pointnclick_day2_trainbetz():
    use prevent_skipping()

This is declared in bug_fixes.rpy.

I've also made an updated CustomSkip() action which is used identically to the Skip action, except it doesn't allow skipping when IsPaCDisabled is False (i.e. PaC is enabled), unless InPaCDialogue is True.

Note that you don't need a custom "are you sure you want to skip" screen - the Skip (and CustomSkip) actions take a `confirm` parameter so you can do `action CustomSkip(confirm=True)` and it will show a confirmation prompt before skipping. I've made this change to the current iterations. common.rpym has the original text as "Are you sure you want to begin skipping?", which I updated to "Start skipping text?\n(You can stop skipping by pressing any button.)" as seen in the original screen. You should be good to delete screen confirmskip unless it serves some other purpose.

The reason the wishlist screen has focus problems is because the imagebuttons are the full size of the screen - this makes it hard for Ren'Py to figure out how to focus from one to the next. Generally it's better to trim them down to only as large as they need to be + position them with x/ypos. With only 2 buttons this isn't a critical performance issue, but it will cause focus problems as you can see. I've trimmed the buttons but left a bit of space for the outline shader.

It looks like IsPaCDisabled is not managed properly resulting in the virtual cursor being onscreen when it shouldn't be.

You can adjust style sigmar in screens_0_GENERAL.rpy
style sigmar:
    font "SigmarOne.otf"
translate japanese style sigmar: # < this is how you change it for different languages
    font "NotoSansCJKjp-Regular.otf"

And use `{=sigmar}Sigmar text{/=sigmar}` instead of `{font=SigmarOne.otf}Sigmar text{/font}`

If there are screens shown with a button, and you don't want to be able to click on the button again to re-show the screen, replace ShowMenu with NoReshowMenu. It does the same thing but makes the button insensitive if it's currently showing.
